# DOCQC-008 桌面首屏视觉清晰度（1440x900）

- 类型: visual
- 优先级: P0
- 关联需求: WTJ-20260703-002 验收标准 5（桌面排版清晰）；需求文档作为沟通工具的首屏说服力
- 前置: `python3 tests/visual/scripts/capture_docs.py`；判读前确认
  `tests/reports/docs-qc/GATE_FAILED` **不存在**（存在=硬门禁失败、截图已被
  移入 `quarantine/`，禁止判读，见 tests/visual/README.md 两级门禁语义）。
  advisory 探针（对比度/alt/console）失败不影响本用例。
- 输入截图: `tests/reports/docs-qc/desktop-first-screen.png`

## 测什么
需求文档在桌面宽度的第一屏是否 30 秒内讲清楚「这是什么产品、给谁看、怎么导航」：
标题/副标题在深色背景上可读、状态 pill 可辨认、导航条完整、首屏无元素重叠或截断。

## 怎么测
1. 运行采集脚本得到 desktop-first-screen.png。
2. 把下面的视觉理解 prompt 和截图交给视觉子代理，收回 JSON 结论。

## 视觉理解 Prompt

```
你在做需求文档静态站的桌面首屏质检。只依据这张 1440x900 首屏截图回答以下具体问题，
不要泛泛描述图片内容：

1. 页面主标题（产品名）是否清晰可读、没有被裁切或与其他元素重叠？
2. 主标题下方的一段副标题文字，在深色背景上是否轻松可读（不眯眼、不费力）？
   引用你实际读出的前 15 个字作为证据。
3. 首屏是否能看到一排页面导航链接（pill/标签形式）？读出其中至少 5 个导航项文字。
4. 首屏右上区域是否有一个说明性文字块（说明本页性质/给谁评审）？它和标题区是否
   有明显重叠、挤压或溢出容器的现象？
5. 首屏整体是否存在任何文字被截断、元素叠压、明显空洞（大面积无内容）或水平滚动迹象？
6. 一个第一次打开该页的评审人，能否只靠首屏判断出「这是一个给3岁孩子的 macOS
   全屏探索应用的需求说明」？指出支撑该判断的可见文字。

输出严格 JSON：
{
  "pass": true/false,
  "reason": "一句话总结判定原因",
  "evidence": ["逐条对应上面问题的具体观察，引用读到的文字"],
  "risks": ["即使 pass 也要列出的潜在视觉风险，没有则空数组"]
}
判定规则：问题 1-5 全部无问题且问题 6 为肯定 → pass=true；任一文字不可读、
截断、重叠或无法完成问题 6 → pass=false。
```

## 怎么算过
视觉子代理返回 `pass=true`，且 evidence 中引用出了真实标题、副标题片段和至少 5 个导航项。
若 pass=false，reason 必须指向具体元素，作为 bug 提报给 TL 卡。
