# DOCQC-010 桌面全页排版密度与章节可视完整性（1440 全页）

- 类型: visual
- 优先级: P1
- 关联需求: WTJ-20260703-002 验收标准 5（桌面排版清晰）+ 验收标准中的十个关键已定点
  需要在页面上"看得见"
- 前置: `python3 tests/visual/scripts/capture_docs.py`；判读前确认
  `tests/reports/docs-qc/GATE_FAILED` **不存在**（存在=硬门禁失败——含 segment
  触顶截断 `segments_complete.truncated=true`，截图已被移入 `quarantine/`，
  禁止判读）。advisory 探针（对比度/alt/console，归属 009/012）失败不影响本用例。
- 输入截图: 桌面全套 segment，**以 probes.json
  `profiles.desktop.screenshots.segments` 清单为唯一权威**，严格按清单顺序
  逐张取图，不要 glob segments 目录（清单才是契约，防止陈旧文件混入）。

## 测什么
1440 宽全页滚动下来，排版是否始终清晰：卡片网格不拥挤不错位、表格列宽合理、
流程步骤完整、无文字溢出容器、无异常大空白；同时以视觉方式确认十个关键主题
（默认界面 / 键盘 / 秘密词 / 五槽 / 问号任务 / 退出机制 / 鼠标反馈 / 奖励 / 素材 /
桌面约束）都有对应的可见章节或版块（内容深度由内容组用例把关，这里只看
"视觉上存在且成块清晰"）。

## 怎么测
把 probes.json segments 清单里的全部桌面 segment 按顺序交给视觉子代理
（同一会话），执行下面 prompt。「全部 segment 逐张判读」是本用例的前提——
segment 触顶截断已被脚本按硬门禁失败处理，不会出现静默缺张。

## 视觉理解 Prompt

```
你在质检一个需求文档静态站的桌面全页排版。下面是按从上到下顺序切开的整页截图。
逐张查看后回答具体问题：

1. 是否存在文字溢出卡片/表格边界、相邻卡片互相叠压、或一行卡片高矮参差导致的
   明显错位？指出所在 segment 编号和大致位置。
2. 是否存在异常空白：某张卡片内部一半以上是空的、或两个章节之间出现约一屏高的
   无内容区域？（正常的章节间距不算。）
3. 页面里的表格：表头和每列内容是否都完整可读？有没有某一列被挤到逐字换行？
4. 页面里的流程步骤条（编号 1-5 的步骤卡）：五步是否都完整显示且顺序可读？
5. 逐张核对以下十个主题是否各自有可见的标题/版块（只判断"看得到这个主题的版块"，
   不判断内容对错）：默认界面（一屏概览/默认画布）、键盘探索、秘密词、五个发现槽、
   问号任务、退出机制、鼠标反馈、奖励系统、素材与声音、桌面运行约束。
   列出每个主题在哪个 segment 看到，缺失的明确写"未见到"。
6. 全页的信息层级是否清楚：章节大标题、卡片小标题、正文三级在视觉上是否能一眼区分？

输出严格 JSON：
{
  "pass": true/false,
  "reason": "一句话",
  "evidence": ["每个问题的结论 + segment 编号定位"],
  "risks": ["不到 fail 程度的拥挤/空洞/层级模糊点"]
}
判定规则：问题 1/3/4 发现任何溢出、叠压、逐字换行的表格列，或问题 5 有主题
"未见到" → pass=false。问题 2/6 的问题记入 risks，除非严重到影响阅读。
```

## 怎么算过
视觉子代理 pass=true，且 evidence 里十个主题全部给出了 segment 定位。
任何主题"未见到"都直接 fail 并同步给内容覆盖用例（去重后归属内容缺失还是排版缺失）。

## 证据
- desktop-seg-*.png 全套
- 视觉 JSON 存 `tests/reports/docs-qc/DOCQC-010.json`

## 鲁棒性说明
- 按"主题可见"而不是"某个 id/锚点存在"判断，TL 换章节结构、合并或改名（例如把
  秘密词和五槽合成一节）时，只要十个主题仍可见即通过——prompt 明确允许主题合并
  在同一版块。
- 不依赖当前 12 栏 grid 实现；判断标准是溢出/叠压/错位这类结果性缺陷。
