设计原则

主线是自由探索

孩子不需要理解任务列表、文字说明或复杂模式。默认状态可以随便按键和移动鼠标，反馈轻、即时、好玩。

已定

大反馈只给有意义事件

秘密词、有效探索里程碑、小任务完成、发现槽全满、连续三次任务完成，才触发明显奖励。乱按也有反馈，但会衰减。

已定

画面必须能回到干净

奖励倾向一次性：烟花、短动画、大贴纸弹出后淡出、临时背景变化。避免永久堆贴纸导致画面越来越乱。

已定

键盘自由探索

普通字母与数字

  • REQ-KB-01 每按一个普通字母，屏幕随机位置弹出这个字母。
  • REQ-KB-02 颜色、大小、旋转、位置都可以变化，但必须保证深色背景上高对比可读。
  • REQ-KB-03 出现方式是“啪”一下弹出，然后约 0.8-1.5 秒逐渐淡出。
  • REQ-KB-04 字母建议用 SVG / Canvas / HTML text 动态生成，不为每种颜色准备贴图。

功能键与重复按键

  • REQ-KB-05 空格、回车可以有轻微波纹或弹跳；Command / Option / Control / Shift 反馈很弱或不计奖励。
  • REQ-KB-06 连续乱按功能键，反馈快速衰减到几乎没有。
  • REQ-KB-07 长按一个键不持续计数。
  • REQ-KB-08 连续重复同一个键超过 3 次后暂停计数；换键后再切回来可以重新计数。
  • REQ-KB-09 正常双写不计入「连续重复同键」的暂停规则，例如连续输入 apple 中的 pp 不会被判定为过度重复（见 REQ-KB-08）。

秘密词

REQ-SEC-01 自由输入

系统监听最近的普通英文字母流，不要求输入框，也不要求回车。

REQ-SEC-02 命中判定

输入字母流中出现完整暗语子串即视为命中；具体边界规则见下方「命中判定规则」。

REQ-SEC-03 出现对象

dog 出现小狗、apple 出现苹果，同时播放授权音效或预生成语音。

命中判定规则

  • REQ-SEC-04 子串命中：输入流中任意连续子串包含完整暗语即触发，如 xxdogxx 中间包含完整 dog 即命中。
  • REQ-SEC-05 重叠触发：如 dogg 在输入到第三个字母 g 时即触发 dog，无需等待独立分隔。
  • REQ-SEC-06 双写不惩罚：如 apple 中的 pp 不会打断或误判命中（词内重复字母不影响暗语子串匹配）。
  • REQ-SEC-07 同词重复命中：同一个词在同一轮内重复命中，只给小反馈，不再点亮新的发现槽（见 REQ-SLOT-01）。同轮已收集词再次命中同样适用本条。
  • REQ-SEC-09 大小写等价：DOG、Dog、dog 均判定为命中同一暗语（PM 已裁决 2026-07-04）。
  • REQ-SEC-10 复合输入顺序命中：如输入 hotdog 过程中可先命中 hot、后命中 dog，两次各自独立触发（PM 已裁决 2026-07-04）。
  • REQ-SEC-11 最长词优先：同一时刻多个后缀同时构成命中时，取最长词优先触发（PM 已裁决 2026-07-04）。

边界规则已由 PM 裁决（2026-07-04）并落文如上，不再列入待确认清单。

发现槽

本章负责发现槽机制本身（点亮规则与来源）；槽满后的宝箱奖励表现细节见 #rewards（REQ-RWD-01）。

发现槽数量口径（PM 已裁决 2026-07-04，WTJ-20260704-083）：发现槽数量是可配置项；默认为 3 个主发现槽（早期文档与 004/010 首次落地曾默认 5，对 3 岁目标用户实测偏多——需集齐 5 次发现才触发一次宝箱，正反馈间隔过长，故 PM 定为默认 3）。数量可按配置扩展至 5；超出 3 个主槽的槽位以 ghost 占位呈现（低透明「可扩展」提示，不参与真实点亮/满槽判定，对应运行版 hud.css .wtj-hud-slot.is-ghost）。下文以「发现槽」泛称，REQ-SLOT-01~04 的语义与编号不变——只是「全部点亮」指的是全部主发现槽。

REQ-SLOT-01 点亮发现槽

秘密词命中或键盘探索里程碑（见 REQ-SLOT-03）会点亮发现槽之一（默认 3 个主发现槽，数量可配置扩展至 5，超出主槽的以 ghost 占位呈现）；当前各格内不重复；同一词重复命中只给小反馈，不再占新格（见 REQ-SEC-07）。

REQ-SLOT-02 开宝箱

所有主发现槽全部点亮后触发宝箱开启，进入一次性大奖励（表现细节见 REQ-RWD-01），随后清空发现槽，进入下一轮。

发现槽来源

  • REQ-SLOT-03 发现槽不只服务秘密词，也可被自由探索里程碑点亮，例如累计 100 次、200 次有效按键。这样孩子即使没有拼出单词，也会因为持续探索获得小目标。
  • REQ-SLOT-04 建议把秘密词命中显示为对应对象图标，把键盘里程碑显示为「键盘星星」这类抽象图标。当前一轮内同类发现不重复。

问号小任务

入口和退出

  • REQ-TASK-01 默认右侧只保留一个低调的问号，不再放 3-4 个图标按钮。
  • REQ-TASK-02 点问号后播放语音任务，不显示中文任务文字。
  • REQ-TASK-03 15 秒未完成：轻提示一次。
  • REQ-TASK-04 30 秒未完成：目标变明显，比如闪一下或稍微放大。
  • REQ-TASK-05 45-60 秒仍未完成：任务自动收起，不算失败（孩子侧退出说明见 #exit / REQ-EXIT-04）。
  • REQ-TASK-06 如果孩子明显转去玩键盘，例如连续 20 个有效键，任务也自动淡出（见 REQ-EXIT-04）。

奖励系统

发现槽全满：开宝箱

  • REQ-RWD-01 宝箱奖励以一次性表现为主，不长期占用屏幕空间；可用烟花、大贴纸弹出后淡出、短动画、临时背景变化、新音效等形式。
  • REQ-RWD-02 宝箱开启后清空发现槽，进入下一轮（见 REQ-SLOT-02）。
  • REQ-RWD-03 烟花建议用 Canvas / SVG 代码生成，预设「满天星、打铁花、圆形、星形」等类型；颜色从少量高质量色板出发做 HSL / HSV 微调，不做完全 RGB 随机。

任务完成：工作状态灯

  • REQ-RWD-04 小任务奖励不与底部发现槽混用；角落放一排很小的工作状态灯，完成一个任务点亮一个。
  • REQ-RWD-05 连续完成 3 个任务，触发「今日工作完成」奖励。
  • REQ-RWD-06 「今日工作完成」奖励表现可以是三个灯一起闪、工作台盖章、小火箭发射、宝箱小开一次；整体风格接近爸爸工作界面的状态灯，但是儿童版。

桌面应用与安全边界

运行环境

  • REQ-DESK-01 目标机器为 2014 款 MacBook Air，应用以 macOS 桌面全屏方式运行。
  • REQ-DESK-02 应用需能打包成可安装/启动形式，例如 DMG 或等价分发方式。
  • REQ-DESK-03 Electron 是否可接受需要 TL 做性能验证；也可以评估更轻的原生方案。需求层只要求孩子能进入一个稳定全屏安全空间。

系统级快捷键限制

  • REQ-DESK-04 Command+Space / Command+Tab 等 macOS 系统级快捷键，普通应用未必能 100% 禁掉。
  • REQ-DESK-05 如需更强控制，需要系统设置、辅助功能权限或家长控制配合，不能在产品需求里虚假承诺。

退出机制

退出机制分三层：误退拦截（阻断快捷键与 Esc 误触）、家长退出（长按 Esc ≥5 秒 + 口令）、孩子侧退出（不设主动退出入口，任务收起不算失败）。

误退拦截

  • REQ-EXIT-01 App 内部拦截 Command+H / Command+W / Command+Q 等常见快捷键，防止孩子误触退出。
  • REQ-EXIT-02 Esc 键不直接触发退出。

家长退出

  • REQ-EXIT-03 家长退出需长按 Esc ≥5 秒，触发口令输入；口令正确后才能退出全屏安全空间。

孩子侧退出

  • REQ-EXIT-04 孩子侧不存在主动退出入口；任务超时自动收起（见 REQ-TASK-05）与转移键盘触发的任务淡出（见 REQ-TASK-06）均不判定为失败，也不触发应用退出。

系统级快捷键限制说明见 #desktop（REQ-DESK-04 / REQ-DESK-05）。

待确认与后续拆卡

待确认

  • 第一版技术壳：Electron、Swift/Objective-C，或其他轻量桌面方案。
  • 2014 MacBook Air 上的性能预算：动画数量、分辨率、音频加载、全屏稳定性。
  • TTS 模型、授权音效来源、缓存策略。
  • 第一批词池和每个词对应素材。
  • 宝箱、贴纸、状态灯的最终视觉风格。

明显不做

  • 默认界面不做复杂工作台、密集面板、很多按钮或收藏栏。
  • 任务不依赖中文文字说明。
  • 不把失败、惩罚、错误提示作为孩子能感知的主体验。
  • 不为了奖励把贴纸永久堆在主画布上。
  • 不承诺普通应用可以完全屏蔽所有 macOS 系统级快捷键。
